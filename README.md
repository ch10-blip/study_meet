# KU-Matching 🐯 (시간표 기반 스터디 매칭 서비스)

고려대학교 학우들의 효율적인 스터디 모집 및 매칭을 위해 제작된 **시간표 기반의 스터디 매칭 플랫폼**입니다.  
기존 에브리타임이나 카카오톡 방에서 일일이 서로의 공강 시간을 조율하던 번거로운 방식을 개선하여, **자신의 빈 시간표만 입력하면 최적의 시간대가 겹치는 스터디 모임을 실시간으로 분석하고 매칭**해 줍니다.

본 프로젝트는 백엔드 서버 없이 브라우저의 `localStorage`만을 이용해 데이터의 저장, 조회 및 가공을 유연하게 수행하도록 개발된 순수 프론트엔드 싱글 페이지 애플리케이션(SPA)입니다.

---

## 주요 기능 및 구현 특징

### 1. 고려대 이메일 기반 재학생 인증 (Simulated)
* `@korea.ac.kr` 도메인 이메일 형식을 필수로 검증합니다.
* 6자리 인증코드를 발송 및 확인하는 절차를 가집니다.
* 영문+숫자 혼합 8자 이상의 강력한 비밀번호 설정 및 학과, 학년 프로필 입력 기능을 갖추고 있습니다.

### 2. 드래그앤드롭 시간표 입력 UI
* 월요일~일요일, 오전 9시부터 오후 10시까지의 그리드 시간표를 제공합니다.
* 마우스 클릭, 드래그 및 모바일 터치 움직임을 통해 연속해서 수업 시간을 파란색으로 선택(토글)할 수 있습니다.
* **선택하지 않은 빈 칸(공강 시간)** 정보가 매칭 데이터로 자동 저장됩니다.

### 3. 실시간 자동 매칭 알고리즘
* 스터디가 개설될 때 등록한 모임 예정 시간과 사용자의 공강 시간 배열의 교집합을 연산합니다.
* `(겹치는 시간 블록 수 / 스터디 요구 시간 블록 수) * 100` 수식에 근거해 실시간 **매칭율(%)**을 산출합니다.
* 탐색 페이지에서 매칭율이 가장 높은 순서대로 스터디 목록이 자동 정렬되어 보여집니다.

### 4. 시각화 분석 및 상세 정보 통제
* 상세 페이지에서 사용자의 시간표와 스터디 모임 시간을 1:1 대조하여 겹치는 슬롯은 **초록색**, 어긋난 슬롯은 **회색**으로 명확히 보여줍니다.
* 스터디장이 신청자(닉네임, 학과, 학년 및 매칭율) 정보를 보고 수락할 시에만 오픈채팅 링크 등의 안전 연락망 정보를 신청한 학우에게 공개합니다.

### 5. 실시간 알림 기능
* 누군가 내 스터디에 참여를 신청했을 때, 혹은 내가 신청한 스터디가 수락/거절되었을 때 실시간으로 시스템 알림이 헤더 영역의 알림함에 등록됩니다.

## 시작하는 방법

서버가 필요 없는 정적 웹 앱으로 즉시 실행이 가능합니다.

### 방법 1. 브라우저에서 직접 열기
* 파일 탐색기에서 `index.html` 파일을 크롬이나 에지 브라우저 등으로 더블 클릭하여 실행합니다.

### 방법 2. Python으로 로컬 서버 띄우기
터미널에서 아래 명령어를 실행하여 구동할 수 있습니다.
```bash
python -m http.server 8000
```
그 후 브라우저에서 `http://localhost:8000`에 접속합니다.

### 방법 3. Node.js로 로컬 서버 띄우기
```bash
npx serve
```
